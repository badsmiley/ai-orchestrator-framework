# Project Template

Стандартная структура для проекта, управляемого через AI-Orchestrator Framework.

## Структура папок

```
{PROJECT}/
├── docs/                   # Документация проекта
│   ├── README.md          # Главный README
│   └── architecture.md    # Архитектура (если нужно)
├── ops/                   # Операционные файлы
│   ├── STATUS.md         # Текущий статус + 3 следующих задачи
│   └── INBOX.md          # Бэклог идей/задач
├── src/                   # Исходный код (если применимо)
├── tests/                 # Тесты (если применимо)
├── .gitignore
├── README.md              # Ссылка на docs/README.md
└── package.json / pyproject.toml / etc
```

## Файлы

### STATUS.md

```markdown
# {PROJECT} — Status

**Обновлено:** YYYY-MM-DD

## Прогресс

- Всего шагов: X
- Завершено: Y

## Текущее состояние

Кратко: что сейчас происходит с проектом.

## Следующие 3 задачи

1. [ ] Задача 1 — статус
2. [ ] Задача 2 — статус
3. [ ] Задача 3 — статус

## Блокировки

- Блокировка 1 (если есть)
```

### INBOX.md

```markdown
# {PROJECT} — Inbox

## Бэклог

- [ ] Идея/задача 1
- [ ] Идея/задача 2

## Готовы к работе

- [ ] Задача, которая ждёт выполнения

## Выполнено

- [x] Задача, которая сделана
```

---

## Micro-report формат (для департаментов)

```markdown
**{PROJECT} — {STATUS}**

✓ Что сделано: ...
→ Что дальше: ...
⚠ Блокировки: ...
```

## Примеры департаментов
